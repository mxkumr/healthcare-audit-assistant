# 2.1 Provider Classification

App: `app/2.1provider-classification/` · Data: CMS Medicare Physician & Other Practitioners (`medicare-ProviderSummary.csv`)

---

## What we're doing

We classify every Medicare provider on **two independent axes** — how much they **cost per patient** and how many **services they bill per patient** — then plot them on a **2-Axis Risk Matrix** for audit triage.

**Example:** For ambulance provider **ABOVE AVERAGE EMS LLC** in Texas, the system flags an extreme billing pattern across four consecutive years. In 2019, they bill a staggering **$20,435.90 per patient** across **1,231 service units per patient**, crushing our outlier thresholds. On a flat spreadsheet, this looks like blatant fraud.

However, by pulling comorbidity data into our drill-down view, the system reveals their small panel of 22 patients carries an unprecedented **11.10 average risk score** with **75% suffering from active diabetes and hypertension**.

Because ambulance "service units" represent cumulative mileage and life-support equipment rather than individual trips, the system proves this isn't fraud — it is highly specialized emergency transport for an exceptionally fragile patient panel. That pattern is what we surface.

**One row in the app** = one provider × year (e.g. 2019 · NPI 1003245093 · ABOVE AVERAGE EMS LLC).

| View | Shows |
|------|-------|
| **Chart (top)** | Stacked column **Risk Matrix** — Cost Classification (X) × Utilization Profile (series) × provider count |
| **Table (bottom)** | Per-provider audit log, grouped by **Provider Name**; expand for year-wise child rows |
| **Object page** | Full risk profile + year-wise history when you navigate into a provider |

**Filters:** Year, State, Specialty, Cost Classification, Utilization Profile

**Example — reading the chart:**

Each stacked bar is a **Cost Classification** band (Highly Efficient → High-Cost Outlier). Colors within the bar show how many providers fall into each **Utilization Profile** (Low / Moderate / High). A tall bar in the top-right cell (High-Cost Outlier + High Utilization) is your primary fraud-triage queue.

**Tip:** Apply a **Year** filter before reading provider counts on the chart — without it, the same NPI counted across multiple years inflates totals.

## The two classifications

Each provider-year gets **two labels** derived from raw CMS totals. Thresholds are fixed dollar/count cutoffs — not peer percentiles.

### Cost Classification (`EfficiencyCategory`)

**What it is:** How much Medicare actually **paid per beneficiary** for this provider in this year.

**Formula:**

```
Cost per Patient = Tot_Mdcr_Pymt_Amt ÷ Tot_Benes
```

**Bands:**

| Label | Threshold |
|-------|-----------|
| **Highly Efficient** | < $150 per patient |
| **Average Spend** | $150 – $899.99 per patient |
| **High-Cost Outlier** | ≥ $900 per patient |

---

### Utilization Profile (`UtilizationCategory`)

**What it is:** How many **service units** the provider bills per beneficiary.

**Formula:**

```
Services per Patient = round(Tot_Srvcs ÷ Tot_Benes)
```

Display is a **whole number**; classification bands use the **raw ratio** before rounding.

**Bands:**

| Label | Threshold |
|-------|-----------|
| **Low Utilization** | < 5 services per patient |
| **Moderate Utilization** | 5 – 14.99 services per patient |
| **High Utilization** | ≥ 15 services per patient |

---

### Reading the matrix together

| Cost ↓ / Utilization → | Low | Moderate | High |
|--------------------------|-----|----------|------|
| **Highly Efficient** | Lean, low-volume | Lean, typical volume | Lean but busy — check panel size |
| **Average Spend** | Typical cost profile | Normal | Busy typical provider |
| **High-Cost Outlier** | Expensive but quiet | Expensive + active | **Top audit priority** — high spend *and* high volume |

## Columns

### Main table

| Column | What it means | How it's calculated |
|--------|---------------|---------------------|
| **Provider Name** | Organization or clinician name | CMS `Rndrng_Prvdr_Last_Org_Name`, uppercased |
| **NPI** | National Provider Identifier | CMS `Rndrng_NPI` |
| **Year** | Calendar year | CMS `Year` |
| **Specialty** | Provider type | CMS `Rndrng_Prvdr_Type` |
| **State** | Two-letter state | CMS `Rndrng_Prvdr_State_Abrvtn` |
| **Patients Served** | Medicare beneficiaries treated | CMS `Tot_Benes`; grouped rows show **peak** annual panel (MAX) |
| **Cost per Patient** | Medicare spend intensity | `Tot_Mdcr_Pymt_Amt ÷ Tot_Benes` |
| **Services per Patient** | Billing volume intensity | `round(Tot_Srvcs ÷ Tot_Benes)`; grouped rows show **highest** annual value (MAX) |
| **Cost Classification** | Cost-efficiency band | Threshold logic above |
| **Utilization Profile** | Service-volume band | Threshold logic above |

### Object page only

These fields are hidden from the main table but appear on **Provider Risk Profile** when you drill in:

| Column | What it means |
|--------|---------------|
| **Avg Patient Age** | Mean age of the beneficiary panel |
| **Avg Risk Score** | Mean HCC risk score — higher = more clinically complex panel |
| **Diabetes %** | CMS-reported diabetes prevalence (often capped at **75%** for small panels) |
| **Hypertension %** | CMS-reported hypertension prevalence (same 75% suppression cap) |

The object page also includes a **Year-wise History** table with the same cost/utilization columns per year.

## Case study — ABOVE AVERAGE EMS LLC (Texas)

Ambulance Service Provider · NPI **1003245093** · flagged **High-Cost Outlier + High Utilization** in every year present (2019–2022).

### 2019 — the headline numbers

Raw CMS row:

| Field | Value |
|-------|-------|
| Location | San Antonio, TX |
| Specialty | Ambulance Service Provider |
| Patients (`Tot_Benes`) | 22 |
| Service units (`Tot_Srvcs`) | 27,084 |
| Medicare paid (`Tot_Mdcr_Pymt_Amt`) | $449,589.85 |
| Avg risk score | 11.10 |
| Diabetes % / Hypertension % | 75% / 75% |

**Derived metrics:**

```
Cost per Patient     = $449,589.85 ÷ 22  = $20,435.90   → High-Cost Outlier (≥ $900)
Services per Patient = round(27,084 ÷ 22) = 1,231         → High Utilization (≥ 15)
```

On a spreadsheet alone, this is an obvious fraud signal. The drill-down tells a different story.

### Why the outlier flags are misleading here

| Signal | What it looks like | What the object page explains |
|--------|-------------------|------------------------------|
| **$20,435 / patient** | Extreme overspending | Tiny 22-patient panel; ambulance trips are high-cost by nature |
| **1,231 services / patient** | Impossible visit count | Ambulance `Tot_Srvcs` = mileage units + equipment charges, not discrete office visits |
| **75% diabetes + 75% hypertension** | Suspiciously round | CMS **privacy suppression cap** for small panels — real comorbidity, masked ceiling |
| **11.10 risk score** | Off the charts | Panel is clinically catastrophic; high transport cost is expected, not fraudulent |

**Audit takeaway:** The matrix correctly surfaces this provider for review. The object page prevents a false-positive fraud referral by showing **clinical context** that raw cost/volume ratios cannot explain alone.

### Four-year pattern (same provider, growing panel)

| Year | Patients | Cost / Patient | Services / Patient | Risk Score | Classification |
|------|----------|----------------|--------------------|------------|----------------|
| 2019 | 22 | **$20,435.90** | **1,231** | 11.10 | High-Cost · High Util |
| 2020 | 26 | $16,383.81 | 892 | 10.14 | High-Cost · High Util |
| 2021 | 67 | $9,640.97 | 528 | 10.72 | High-Cost · High Util |
| 2022 | 14 | $13,112.68 | 723 | 8.43 | High-Cost · High Util |

Even as the panel grows and per-patient cost drops, every year stays firmly in the top-right matrix cell. That consistency is the audit signal; the comorbidity drill-down is the exoneration path.

## Worked example — building one classification

**Provider:** ABOVE AVERAGE EMS LLC · **2019**

**Step 1 — Cost per Patient**

```
$449,589.85 ÷ 22 = $20,435.90
```

$20,435.90 ≥ $900 → **High-Cost Outlier**

**Step 2 — Services per Patient**

```
27,084 ÷ 22 = 1,231.09 → round → 1,231
```

1,231.09 ≥ 15 → **High Utilization**

**Step 3 — Matrix placement**

```
High-Cost Outlier × High Utilization → top-right audit cell
```

**Step 4 — Drill-down before escalating**

Open object page → confirm risk score 11.10, diabetes/hypertension 75%/75%, specialty = Ambulance → contextualize before referral.

## Source data

Each CSV row = one provider × year. Classifications are computed in the `ProviderCostEfficiency` view — no manual tagging.

| CMS field | Used for |
|-----------|----------|
| `Year` | Time dimension |
| `Rndrng_NPI` | Provider key |
| `Rndrng_Prvdr_Last_Org_Name` | Provider name |
| `Rndrng_Prvdr_Type` | Specialty |
| `Rndrng_Prvdr_State_Abrvtn` | State |
| `Tot_Benes` | Patients served; divisor for per-patient metrics |
| `Tot_Srvcs` | Service units; divisor for utilization |
| `Tot_Mdcr_Pymt_Amt` | Medicare paid; numerator for cost per patient |
| `Bene_Avg_Age` | Object page — panel age |
| `Bene_Avg_Risk_Scre` | Object page — clinical complexity |
| `Bene_CC_PH_Diabetes_V2_Pct` | Object page — diabetes prevalence |
| `Bene_CC_PH_Hypertension_V2_Pct` | Object page — hypertension prevalence |
