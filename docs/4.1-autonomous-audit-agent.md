# 4.1 Autonomous Audit Agent (Joule-Ready)

App: `srv/medicare-service.cds` + `srv/medicare-service.js` · Data: Tasks 1–3 analytical views

Part of **Task 4 — Autonomous Audit Agent**. Exposes OData V4 **actions** that Joule Studio can register as capabilities, plus a **scratchpad entity** for step-by-step reasoning logs.

---

## What we're doing

We ask: *"Can Joule invoke our CAP backend to run a multi-slice Medicare audit and return a structured compliance narrative?"*

```
Joule prompt + optional filters (year, state, specialty, NPI)
         │
         ▼
MedicareService.investigateAnomalies  (or focused tool actions)
         │
         ├── RiskCostVolumeDynamics      (Task 3.1)
         ├── PlaceOfServiceAnalysis      (Task 3.2)
         ├── CredentialDiscrepancies     (Task 3.3)
         ├── CostAnalysisV2              (Task 1.1 regional)
         ├── ProviderCostEfficiency      (Task 2.1)
         ├── SpecialtyPeerDeviations     (Task 2.2)
         └── ServiceDetails              (HCPCS drill-down)
         │
         ▼
AgentScratchpad (reasoning log) + Markdown narrative + flagged NPIs
```

**Year handling:** All actions accept an optional `year` parameter. If omitted or invalid, the agent resolves to **`2022`** when present, otherwise the latest year in the dataset (`listAuditYears` returns `2019`–`2023` in the CMS sample).

## Joule action catalog

| Action | Purpose | Key parameters |
|--------|---------|----------------|
| **`investigateAnomalies`** | Primary orchestrator — full multi-slice report | `prompt`, `year`, `state`, `specialty`, `npi` |
| **`getRegionalBillingOutliers`** | Task 1.1 regional hotspot tool | `state` (required), `year` |
| **`getProviderClaimDetails`** | Task 2.1 + ServiceDetails drill-down | `npi` (required), `year` |
| **`getSpecialtyPeerOutliers`** | Task 2.2 peer deviation tool | `specialty`, `year`, `state` |
| **`listAuditYears`** | Discovery — available CMS years | _(none)_ |

### Entity for traceability

| Entity | Purpose |
|--------|---------|
| **`AgentScratchpad`** | Read-only OData log of each tool invocation (sessionId, step, input/output JSON) |

## OData endpoints (local dev)

Base URL: `http://localhost:4004/medicare`

### List available years

```bash
curl -X POST http://localhost:4004/medicare/listAuditYears \
  -H "Content-Type: application/json" -d '{}'
```

### Full investigation (PIZZO-BERKEY demo)

```bash
curl -X POST http://localhost:4004/medicare/investigateAnomalies \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Investigate CA Pain Management billing anomalies",
    "year": "2022",
    "state": "CA",
    "specialty": "Pain Management",
    "npi": "1003056821"
  }'
```

**Response fields:**

| Field | Type | Description |
|-------|------|-------------|
| `narrative` | String | Markdown compliance report |
| `confidenceScore` | Integer | 70–95 based on data slices populated |
| `year` | String | Resolved performance year |
| `flaggedNPIs` | String | JSON array of high-cost outlier NPIs from `ProviderCostEfficiency` |
| `reasoningSteps` | String | JSON array of tool steps executed |
| `sessionId` | String | Correlates with `AgentScratchpad` rows |

### Regional tool

```bash
curl -X POST http://localhost:4004/medicare/getRegionalBillingOutliers \
  -H "Content-Type: application/json" \
  -d '{"state": "FL", "year": "2022"}'
```

### Provider tool

```bash
curl -X POST http://localhost:4004/medicare/getProviderClaimDetails \
  -H "Content-Type: application/json" \
  -d '{"npi": "1003056821", "year": "2022"}'
```

### Read scratchpad

```bash
curl "http://localhost:4004/medicare/AgentScratchpad?\$top=10&\$orderby=createdAt%20desc"
```

## Joule Studio registration

1. **Register OData capability** pointing to `/medicare` (via approuter destination `srv-api` in BTP).
2. **Import actions** from `$metadata` — descriptions come from `@Core.Description` annotations on each action.
3. **Recommended tool mapping:**

| Joule intent | CAP action |
|--------------|------------|
| "Run audit" / "investigate anomalies" | `investigateAnomalies` |
| "Why is {state} expensive?" | `getRegionalBillingOutliers` |
| "Profile NPI {npi}" | `getProviderClaimDetails` |
| "Who deviates from peers in {specialty}?" | `getSpecialtyPeerOutliers` |
| "What years are available?" | `listAuditYears` |

4. **Security:** Assign role **`audit_analyst`** (`$XSAPPNAME.audit.read`) for OData read + action invocation.
