# 2.2B Entity Type Comparison

App: `app/2.2borganization-profiling/` · Data: CMS Medicare Physician & Other Practitioners (`medicare-ProviderSummary.csv`)

Builds on **Task 2.1** (`ProviderCostEfficiency`) — same per-provider cost and utilization metrics, now split by **CMS entity type**: **Individual clinicians** vs **Corporate organizations**.

---

## What we're doing

We ask: *"Do organizations or individual clinicians charge more per patient — and where does that show up by specialty?"*

**2.1** classifies each provider on fixed cost/utilization thresholds. **2.2** compares providers to specialty peers. **2.2B** adds a **macro entity lens** using CMS field `Rndrng_Prvdr_Ent_Cd`:

| CMS code | Entity class |
|----------|--------------|
| `I` | Individual Clinician |
| `O` | Organization / Corporate Network |

**Example (2022):** Organizations average **$541.90 per patient** vs **$308.96** for Individual clinicians — a **75.4% cost premium**. The KPI band names **Organization / Corporate Network** as the likely higher-charging entity. Expand **Medical Oncology** in the table to see 30 individual clinicians (e.g. BATTY, BUCK, KUMARAN) with per-provider cost side by side.

**One chart row** = one entity class × year (2 bars). **One table row** = one provider × year, grouped under **Specialty**.

## How the math works

### Step 1 — Entity type label (`ProviderCostEfficiency`)

Each provider-year inherits entity type from CMS:

```
EntityTypeCode = Rndrng_Prvdr_Ent_Cd

EntityType =
  'O' → Organization / Corporate Network
  'I' → Individual Clinician
  else → Unknown Entity Type
```

Cost and utilization metrics are the same formulas as Task 2.1:

```
Cost per Patient     = Tot_Mdcr_Pymt_Amt ÷ Tot_Benes
Services per Patient = round(Tot_Srvcs ÷ Tot_Benes)
```

---

### Step 2 — Macro comparison (`EntityTypeComparisons`)

One row per **Year + Entity Type** (aggregated from all providers in that class):

```
Total Unique Providers       = count(distinct NPI)
Total Patients Served        = sum(Tot_Benes)
Macro Avg Cost per Patient   = avg(Cost per Patient)
Macro Avg Services per Patient = avg(Services per Patient)
High-Cost Outlier Count      = sum(High-Cost Outlier flags from 2.1)
High Utilization Outlier Count = sum(High Utilization flags from 2.1)
```

**2022 macro snapshot:**

| Entity Type | Providers | Patients | Avg Cost/Patient | High-Cost Outliers | High Util Outliers |
|-------------|----------:|---------:|-----------------:|-------------------:|-------------------:|
| Individual Clinician | 9,524 | 2,566,244 | **$308.96** | 504 | 977 |
| Organization / Corporate Network | 476 | 252,249 | **$541.90** | 52 | 54 |

Organizations charge more per patient on average, but Individuals contribute far more **absolute** outlier volume (504 vs 52) because there are ~20× more individual NPIs in this sample.

---

### Step 3 — KPI insight (`EntityTypeCostInsight`)

One row per **Year** — joins Organization and Individual macro rows:

```
HigherChargingEntity =
  entity class with greater MacroAvgCostPerPatient

CostPremiumPct =
  round(|Org avg − Individual avg| ÷ lower avg × 100, 1)
```

**2022 worked math:**

```
Higher entity avg  = $541.90  (Organization)
Lower entity avg   = $308.96  (Individual)

CostPremiumPct = (541.90 − 308.96) ÷ 308.96 × 100 = 75.4%
```

## 2.1 vs 2.2 vs 2.2B — when to use which

| Question | Use |
|----------|-----|
| "Is this provider above $900/patient?" | **2.1** fixed Cost Classification |
| "Is this provider abnormal for their specialty?" | **2.2** peer deviation |
| "Do organizations or individuals charge more nationally?" | **2.2B** macro entity chart + KPIs |
| "Within Medical Oncology, who are the expensive individuals?" | **2.2B** table — expand specialty group |
| "On the **same HCPCS code**, who charges more?" | Not this task — needs service-line comparison |

**Audit caveat:** Macro entity comparison is **not apples-to-apples** across specialties. Organizations and Individuals often bill different specialty mixes. Use 2.2B for **directional** entity signals; drill the specialty table before escalating a corporate network.

## Columns

### KPI band (`EntityTypeCostInsight`)

| Field | What it means |
|-------|---------------|
| **Likely Higher-Charging Entity** | Organization or Individual with higher macro avg cost |
| **Entity Cost Premium (%)** | % gap between higher and lower entity class |
| **Higher Entity Avg Cost** | Macro $/patient for the winning class (OData only) |
| **Lower Entity Avg Cost** | Macro $/patient for the lower class (OData only) |

### Main table (`EntityTypeProviderProfiles`)

| Column | What it means | How it's calculated |
|--------|---------------|---------------------|
| **Specialty** | CMS provider type | `Rndrng_Prvdr_Type`; table **groups** by this field |
| **Provider Name** | Organization or clinician | CMS name, uppercased; visible when group is **expanded** |
| **Entity Type** | Individual vs Organization | Mapped from `Rndrng_Prvdr_Ent_Cd` |
| **NPI** | National Provider Identifier | CMS `Rndrng_NPI` |
| **Cost per Patient** | Medicare spend intensity | `Tot_Mdcr_Pymt_Amt ÷ Tot_Benes` |
| **Services per Patient** | Billing volume intensity | `round(Tot_Srvcs ÷ Tot_Benes)` |
| **Cost Classification** | 2.1 cost band | Highly Efficient / Average Spend / High-Cost Outlier |
| **Utilization Profile** | 2.1 utilization band | Low / Moderate / High Utilization |

### Macro view (`EntityTypeComparisons` — OData / Task 2 overview card)

| Column | What it means |
|--------|---------------|
| **Entity Type** | Individual Clinician or Organization / Corporate Network |
| **Unique Providers** | Distinct NPI count in entity class |
| **Total Patients Served** | Sum of beneficiaries |
| **Avg Cost per Patient** | Macro mean cost intensity |
| **High-Cost Outlier Count** | Providers flagged ≥ $900/patient |
| **High Utilization Outlier Count** | Providers flagged ≥ 15 services/patient |

## Worked example — Medical Oncology (2022)

**Macro context:** All 30 Medical Oncology providers in 2022 are **Individual Clinicians** — no Organization rows in this specialty for that year. You cannot compare I vs O *within* Medical Oncology in this sample; you compare individuals against the national org-vs-ind macro KPIs instead.

**Expanded table sample:**

| Provider | Entity Type | Cost/Patient | Classification |
|----------|-------------|-------------:|----------------|
| BATTY | Individual Clinician | $245.19 | Average Spend |
| BUCK | Individual Clinician | $618.99 | Average Spend |
| KUMARAN | Individual Clinician | $326.55 | Average Spend |

**Audit takeaway:** Specialty drill-down proves *who* drives oncology spend. Macro KPIs prove *which entity class* charges more portfolio-wide. Use both layers together.

---

## Dataset caveats

- This project uses a **50,000-row sample** of CMS Medicare data (2019–2023), not the full national file.
- Some specialties have **only Individual** or **only Organization** providers in a given year — entity comparison within that specialty may be impossible.
- Macro entity averages mix **all specialties** — always drill the table before attributing overcharging to corporate networks.
