# 1.2 Rural Analysis

App: `app/1.1rural-analysis/` · Data: CMS Medicare Physician & Other Practitioners (`medicare-ServiceDetails.csv`)

---

## What we're doing

We compare **how aggressively providers bill Medicare** across **geographic tiers** — Urban, Suburban, and Rural — for the same medical procedure.

**Example:** For procedure **J7613** (Albuterol), a rural provider in Kayenta, AZ bills $12.94 per unit while suburban providers in Tifton, GA bill $2.00 per unit. Both get rejected at ~99%, but rural rejects **slightly more** than the procedure average (+0.84% tier deviation). That pattern is what we surface.

**One row in the app** = one procedure code in one geographic tier (e.g. J7613 in Rural / Isolated).

---

## What you see

| View | Shows |
|------|-------|
| **Chart (top)** | Procedures with the biggest **Tier Deviation** — grouped bars per Urban / Suburban / Rural |
| **Table (bottom)** | Full numbers per procedure × tier: volume, dollars, rejection rate, deviation |

**Filters:** Procedure Code, Structural Tier

**Example — reading the chart:**

For J7613 you would see three bars side by side:
- Rural bar slightly **above** zero (+0.84%)
- Urban bar slightly **below** zero (−0.15%)
- Suburban bar further **below** zero (−0.85%)

The chart highlights *relative* differences between tiers, not the raw 99% rejection rate (which is high everywhere for this code).

---

## Geographic tiers

Provider location (RUCA code) maps to a tier:

| RUCA | Tier | Example from dataset |
|------|------|----------------------|
| 1 – 3 | Urban / Metro | Annandale, VA (RUCA 1) |
| 4 – 6 | Suburban / Micro | Tifton, GA (RUCA 4) |
| 7 – 10.3 | Rural / Isolated | Kayenta, AZ (RUCA 7) |

Only procedures that appear in **at least 2 tiers** are shown, so we can actually compare them.

**Example:** J7613 appears in all three tiers → included. A procedure billed only in urban areas → excluded (nothing to compare).

## Columns

| Column | What it means | How it's calculated |
|--------|---------------|---------------------|
| **Procedure Code** | HCPCS billing code | From CMS data |
| **Procedure Description** | What the code represents | From CMS data |
| **Structural Tier** | Urban, Suburban, or Rural | From provider RUCA code |
| **Frequency** | How many services were billed | Sum of service counts |
| **Total Billed Charges** | What providers asked Medicare to pay | Avg submitted charge × services, summed |
| **Rejected Over-Charges** | What Medicare refused to allow | Total Billed − Total Allowed |
| **Rejection Rate (%)** | Share of the bill that was rejected | Rejected ÷ Billed × 100 |
| **Procedure Baseline (%)** | Normal rejection rate for this procedure | Weighted average across all tiers for that code |
| **Tier Deviation (%)** | How far this tier is from normal | Rejection Rate − Procedure Baseline |
| **Total Medicare Paid** | Cash Medicare actually sent out | Avg payment × services, summed |

## Examples — how each column is built

All examples use **J7613, Rural / Isolated, Kayenta AZ 2019** unless noted.

### From one CSV row to one tier row

Raw CMS row:

| Field | Value |
|-------|-------|
| Location | Kayenta, AZ |
| RUCA | 7 → **Rural / Isolated** |
| Services | 68 |
| Avg Submitted | $12.94 |
| Avg Allowed | $0.04 |
| Avg Paid | $0.02 |

This is one of nine J7613 rows. The app sums all rows in the same tier before calculating rates.

---

### Frequency

```
Rural frequency = 68 services (Kayenta AZ only)
```

Urban adds Annandale (19) + Tulsa (63) + Nashville (36) = **118**
Suburban adds five Tifton GA years = **406**

---

### Total Billed Charges

```
Rural = $12.94 × 68 = $879.92
```

Urban = ($5.00×19) + ($4.00×63) + ($0.42×36) = **$362.12**
Suburban = ($2.00×77) + ($2.00×39) + … = **$812.00**

---

### Rejected Over-Charges

Medicare allows $0.04/unit for every J7613 row.

```
Rural allowed  = $0.04 × 68  = $2.72
Rural rejected = $879.92 − $2.72 = $877.20
```

Rejected = the gap between what was **billed** and what Medicare **allows**.

---

### Rejection Rate

```
Rural rejection rate = ($877.20 ÷ $879.92) × 100 = 99.69%
```

Same formula per tier:

| Tier | Rejected | Billed | Rate |
|------|----------|--------|------|
| Rural | $877.20 | $879.92 | 99.69% |
| Urban | $357.40 | $362.12 | 98.70% |
| Suburban | $795.76 | $812.00 | 98.00% |

---

### Procedure Baseline

Average rejection for J7613 **across all tiers combined**:

```
Baseline = ($2,030.36 ÷ $2,054.04) × 100 = 98.85%
```

This number is the **same on every tier row** for J7613 — it's the procedure's normal rejection level.

---

### Tier Deviation

```
Rural deviation  = 99.69% − 98.85% = +0.84%   (worse than average)
Urban deviation  = 98.70% − 98.85% = −0.15%   (slightly better)
Suburban deviation = 98.00% − 98.85% = −0.85% (most compliant)
```

| Value | Meaning |
|-------|---------|
| **Positive** (+) | This tier rejects **more** than the procedure average |
| **Negative** (−) | This tier is **more compliant** than average |
| **Near zero** | In line with other tiers |

Color coding: green = 0–5 pp, yellow = 5–15 pp, red = > 15 pp.

---

### Total Medicare Paid

```
Rural paid = $0.02 × 68 = $1.36
```

**Do not confuse with rejected charges:**

| Column | Rural J7613 | What it tells you |
|--------|-------------|-------------------|
| Rejected Over-Charges | **$877.20** | Medicare disallowed almost the entire claim |
| Total Medicare Paid | **$1.36** | Medicare only paid $0.02 per unit |

High rejection + tiny payout happens when Medicare's fee schedule is very low ($0.04 allowed, $0.02 paid).

## Full example — J7613 summary table

| Column | Rural | Urban | Suburban |
|--------|-------|-------|----------|
| Frequency | 68 | 118 | 406 |
| Total Billed | $879.92 | $362.12 | $812.00 |
| Rejected Over-Charges | $877.20 | $357.40 | $795.76 |
| Rejection Rate | 99.69% | 98.70% | 98.00% |
| Procedure Baseline | 98.85% | 98.85% | 98.85% |
| **Tier Deviation** | **+0.84%** | −0.15% | −0.85% |
| Total Medicare Paid | $1.36 | $3.54 | $12.18 |

**Audit takeaway:** Rejection is ~99% everywhere, but rural Kayenta bills **$12.94/unit** vs suburban Tifton at **$2.00/unit**. Rural tier deviation (+0.84%) flags that rural providers push harder against the fee cap than other tiers for this code.

## More examples from the dataset

### 64495 — large negative deviation

Suburban providers bill much **less** aggressively than the procedure average:

```
Suburban rejection rate = 0.29%
Procedure baseline      = 76.67%
Tier deviation          = 0.29% − 76.67% = −76.38%
```

A big negative number means that tier is far **more compliant** than other tiers for that procedure — worth investigating why suburban looks so different.

### 00732 — why parsing matters

One rural row had `Tot_Srvcs = "1,200"` in the CSV. Without stripping the comma, the system read it as **1** service instead of 1,200 — understating billed charges. The model removes commas before math so totals stay correct (~$43,545 billed rural).

## Source data

Each CSV row = one provider × year × procedure. We aggregate by procedure code + tier before calculating columns.

| CMS field | Used for |
|-----------|----------|
| `HCPCS_Cd` / `HCPCS_Desc` | Procedure code and name |
| `Rndrng_Prvdr_RUCA` | Geographic tier |
| `Tot_Srvcs` | Service count |
| `Avg_Sbmtd_Chrg` | Billed charges |
| `Avg_Mdcr_Alowd_Amt` | Allowed amount (for rejections) |
| `Avg_Mdcr_Pymt_Amt` | Actual Medicare payment |
